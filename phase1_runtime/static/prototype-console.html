Capability Storyboard

从回答问题，走向沉淀解法

这个页面不是主工作台，而是能力展板。它用几个典型 flow，把系统从“理解文档”到“完成求解”，再到“沉淀方法”的能力边界串起来。

核心目标：沉淀解法 已有方法处理方式 进入内部处理方式

演示重点

理解问题 → 组织解法 → 形成答案

有现成方法时复用已有能力；没有现成方法时，继续进入探索、试跑和人工审核闭环。

系统核心能力

方法库

沉淀一类问题该怎么解，而不是只保存一次性答案。

文档理解

读懂 PDF，并围绕当前问题形成可推理的上下文。

上下文组织

把问题、文档理解和业务语义整理成后续可执行的求解视图。

方法匹配

先召回，再验证当前问题适合哪种解法。

方法草稿生成

把当前问题、上下文和已有方法组合成一次可执行的草稿。

增强求解

按草稿执行代码、工具和答案组织，把解法真正跑完。
Existing Method

有现成方法时怎么走

问题与材料

用户上传金融文档并提出业务问题。

理解文档

系统围绕当前问题读取文档，并聚焦更相关的证据。

匹配已有方法

从方法库中召回候选解法，并验证是否真的适配当前问题。

形成方法草稿

把已有方法和当前上下文整理成本次可执行的解法草稿。

完成求解

按草稿执行代码、工具和答案组织。

沉淀结果

输出用户答案，并把本次求解继续沉淀下来。
New Method

没有现成方法时怎么走

暂未命中现成方法

当方法库中没有合适解法时，系统不会直接失败退出。

生成候选解法

系统根据问题、上下文和已有经验提出新的解法候选。

试跑新解法

把候选解法继续转成可执行草稿，并尝试解决当前问题。

人工审核

判断这套解法是否通用、是否值得留下，不通过则继续迭代。

进入方法库

通过后才真正成为长期可复用的方法。
