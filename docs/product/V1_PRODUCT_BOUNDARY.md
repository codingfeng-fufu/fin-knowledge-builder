# V1 产品边界

> 目的：明确第一版到底做什么、不做什么、向谁承诺什么。
> 结论先行：V1 不应该把自己包装成完整“方法资产平台”，而应该先落成一个可用的“带证据的金融文档求解工作台”。

## 1. V1 产品定义

### 对外定义

V1 对外定义为：

**带证据的金融文档求解工作台**

它的核心承诺是：

- 用户上传金融文档并提问
- 系统给出答案或明确状态
- 系统展示关键依据
- 当无法稳定回答时，明确进入人工复核

### 对内定义

V1 的内部支撑系统仍然是：

**把答案沉淀成方法的规则资产平台**

但这层能力主要服务于：

- 系统学习
- 方法治理
- 内部审核

不应该在 V1 里直接成为普通用户的主价值表达。

## 2. V1 要解决的问题

V1 只解决两个问题：

1. 如何让用户更快拿到一个可辩护的答案。
2. 如何让系统在答不稳时，把问题移交给内部方法沉淀流程。

如果一个功能不直接服务这两个问题，就不应该优先进入 V1。

## 3. V1 明确纳入范围的内容

### 3.1 `/workspace` 作为唯一主入口

V1 的主入口只保留 `/workspace`。

它负责：

- 上传文档
- 输入问题
- 触发求解
- 展示答案
- 展示关键证据
- 展示当前状态

### 3.2 三种用户可理解的状态

V1 面向用户只应该稳定表达三类状态：

- 已生成结论
- 需要补充材料
- 需要人工复核

不要让用户去理解更底层的：

- direct_match
- rule_composable
- exploration
- binding_status

这些可以保留给专家视图和内部调试。

### 3.3 稳定场景优先

V1 只承诺少数稳定场景。

当前建议优先承诺：

- `fund_nav_warning`

原因很简单：

- 这条链更稳定
- 可解释性更强
- 更适合做首次体验

### 3.4 允许内部自动沉淀，但不把沉淀动作外显为用户主流程

exploration 触发后，系统内部仍然可以自动生成：

- feedback
- draft
- review

但对普通用户来说，主表达只应该是：

- 当前问题需要人工复核
- 系统已进入内部处理流程

### 3.5 `/ops` 保留为内部页面

V1 中 `/ops` 是内部治理后台，不是普通用户产品页。

它继续承担：

- review
- publish
- rollback
- asset view

## 4. V1 明确排除的内容

### 4.1 不把“多智能体生规则”当成外部卖点

这是内部能力，不是 V1 外部承诺。

原因：

- 用户不一定关心
- 它无法直接提升首次可感知价值
- 它反而会让产品看起来更像实验系统

### 4.2 不把 Rule Graph 暴露给普通用户

图谱、社区、meta rule 这些内容暂时只对内部治理有价值。

### 4.3 不把 `/workflow` 当成主产品流程

`/workflow` 有价值，但它更像：

- 演示页
- 解释页
- 内部说明页

不是用户完成任务的主入口。

### 4.4 不承诺“通用金融场景都能处理”

V1 不应该使用过宽口径。

应该只承诺：

- 若干已验证稳定的场景
- 带证据的问题求解能力

### 4.5 不让普通用户直接接触 publish / rollback

这属于平台治理，不属于业务用户日常任务。

## 5. V1 默认体验建议

### 5.1 默认样本

建议默认样本回切到：

- `fund_docx_direct_warn`

而不是当前更重、更不稳定的 `equity_research_h3_code_upside_calc`。

### 5.2 页面文案

首页文案可以保留“把答案变成方法”的总方向。

但主按钮和主流程文案应优先强调：

- 上传文档
- 提出问题
- 获取答案与依据

### 5.3 不再自动执行默认样本

当前前端已经不自动执行，这个方向是对的。

V1 应继续保留手动触发。

因为用户要清楚知道：

- 什么时候发起了一次正式求解
- 当前结果是否来自真实输入

## 6. V1 的验收标准

如果按产品视角定义 V1，至少要满足：

1. 在 golden cases 上，`direct_match` 能稳定给出可复核答案。
2. 用户能明显区分“已回答”和“需要人工复核”。
3. exploration 触发后，内部能自动形成 review 对象。
4. `/workspace` 的首次体验不依赖理解规则平台概念。
5. 页面文档和真实交互行为一致。

## 7. V1 之后再谈什么

以下内容建议放到 V1 之后：

- 把方法资产平台单独产品化
- 把 `workflow` 正式做成对外页面
- 扩展更多金融场景
- 对外讲多智能体规则生成
- 做用户可见的规则管理能力

## 8. 一句话边界

V1 不是“一个完整的金融规则资产平台产品”。

V1 是：

**一个由规则资产平台支撑的、带证据的金融文档求解工作台。**
